# TLS-сертификаты для GigaChat

Эмбеддинги GigaChat (Sber) работают по TLS, подписанному корневым УЦ Минцифры.
Чтобы Python доверял этому соединению, положите сюда корневой сертификат:

    certs/russian_trusted_root_ca.pem

Этот файл монтируется в контейнер как
`/workspace/certs/russian_trusted_root_ca.pem` (см. `docker-compose.yml`) и
указывается в `.env` через `GIGACHAT_CA_BUNDLE`.

## Где взять
Корневой сертификат «Russian Trusted Root CA» публикуется Минцифры
(портал Госуслуг). Скачайте его на хосте с доступом и перенесите в закрытый
контур вместе с этим пакетом. Если в корпоративном контуре свой GigaChat
on-prem — используйте сертификат вашего внутреннего УЦ и пропишите
эндпоинты `GIGACHAT_BASE_URL` / `GIGACHAT_OAUTH_URL` в `.env`.

## Без сертификата
Если CA-бандла нет, можно временно отключить проверку TLS в `.env`:

    GIGACHAT_VERIFY_SSL=false

Это небезопасно (MITM) — только для отладки, не для промышленной эксплуатации.

> Сами файлы сертификатов в git не коммитятся (см. `.gitignore`).
